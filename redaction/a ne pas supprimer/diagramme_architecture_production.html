Légende des Composants

  • Utilisateurs (Bleu): Interface web, clients API, dashboard administratif
  • Zone 4 - Sécurisation Production (Orange):
    • WAF: Rate limiting 100 req/min par IP, détection SQL injection/XSS
    • JWT: Access token (15min) + Refresh token (7j), algorithme RS256
    • RBAC: 3 rôles (admin, agent, guest), matrice de permissions
    • Anomaly Detection: Isolation Forest sur features requête, seuil risque 0.7
  • APIs FastAPI (Vert): 2 instances dédiées (baseline + secured), asynchrone Uvicorn
  • Modèles PyTorch (Violet): Baseline non-sécurisé vs Secured (adversarial training + chiffrement AES-256-GCM)
  • Zone 5 - Gouvernance & Audit (Rose):
    • Audit Logger: Logs immuables PostgreSQL, hash chain SHA-256, pseudonymisation RGPD
    • Prometheus: Métriques temps réel (requêtes/sec, latence, accuracy), scrape 15s
    • Grafana: 5 dashboards (Overview, Security, Models, Anomalies, Audit), alerting
    • Application Logs: JSON structuré, rotation 7j, agrégation Loki optionnelle
  • Stockage Persistant (Turquoise): PostgreSQL (users, audit, WAF), Redis (sessions JWT, rate limiting, cache)

Flux de Données Principal

  1. Requête utilisateur → Nginx HTTPS (port 443)
  2. Zone 4 - Sécurisation:
    • WAF vérifie rate limit (100 req/min) et patterns malveillants
    • JWT valide le token (signature RS256, expiration)
    • RBAC vérifie les permissions selon le rôle
    • Anomaly Detection calcule risk score (seuil 0.7)
  3. Routage → API Baseline (8001) ou Secured (8002) selon endpoint
  4. Inférence → Modèle PyTorch (déchiffrement si secured)
  5. Zone 5 - Audit:
    • Log immuable PostgreSQL (timestamp, IP, user_id, prediction, confidence, duration)
    • Métriques Prometheus (compteurs, histogrammes, gauges)
    • Logs applicatifs JSON (rotation 7j)
  6. Réponse JSON → Utilisateur via Nginx
